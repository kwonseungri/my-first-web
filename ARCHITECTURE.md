# 프로젝트 아키텍처 및 설계

## 1. 프로젝트 목표
간단하고 직관적인 개인 블로그 구축. 
독자는 글을 편하게 읽고, 작성자는 글을 쉽게 추가 및 관리할 수 있습니다.

## 2. 페이지 맵 (URL 구조)
- **홈 (`/`)**: 블로그 소개 메인 화면
- **글 목록 (`/posts`)**: 작성된 게시글 리스트 조회
- **글 상세 (`/posts/[id]`)**: 단일 게시글 내용 조회
- **글 작성 (`/posts/new`)**: 신규 게시글 추가 폼
- **로그인 (`/login`)**: (향후 추가 예정) 사용자 인증
- **마이페이지 (`/mypage`)**: (향후 추가 예정) 사용자 정보 및 작성 글 관리

## 3. 유저 플로우
- **글 읽기**: 홈 → 블로그(목록) 진입 → 목록 중 하나 클릭 → 상세 페이지 조회 → (필요시) 목록으로 돌아가기
- **글 작성**: 블로그 메뉴 → '새 글 쓰기' 진입 → 폼에 제목/내용 입력 → '저장하기' 클릭 → 글 목록으로 자동 리다이렉트 (추후 로그인 제어 포함)

## 4. 컴포넌트 구조 (shadcn/ui 기반)
- **공통 Layout**: 내비게이션 바 및 하단 푸터 영역
- **Card**: 게시글 목록 내 개별 아이템 디자인용
- **Button**: 클릭 이동, 제출(저장), 삭제 등의 동작 요소
- **Input / Textarea**: 새 글 작성 폼 요소 및 검색 필드
- **Dialog**: 삭제 및 오류 확인 시 경고창 팝업용

## 5. 데이터 모델 (추후 Supabase 연동용)
```
users (사용자)
├── id: UUID (PK)
├── email: string
├── name: string
└── role: 'user' | 'admin'

posts (포스트)
├── id: bigint (PK)
├── title: string
├── content: text
├── author_id: UUID (users 참조)
└── created_at: timestamp
```
